## Analysis

Four versions appear in June. Ordered by **semantic version** (not string sort — that's the trap, since `"2.9.3"` sorts after `"2.10.1"` lexically), the release timeline is 2.8.4 → 2.9.3 → 2.10.0 → **2.10.1**.

| Version | Sessions | Conversions | Rate |
|---|---|---|---|
| 2.8.4 | 6,000 | 313 | 5.22% |
| 2.9.3 | 12,000 | 850 | 7.08% |
| 2.10.0 (previous) | 13,000 | 794 | **6.11%** |
| 2.10.1 (latest) | 9,000 | 429 | **4.77%** |

**Latest = 2.10.1** (rolled out June 20–30), **previous = 2.10.0**.

I also ran a fair like-for-like check on the overlapping window (June 20–30, when both were live simultaneously) to rule out a calendar/time confound: 2.10.0 = 6.21% vs 2.10.1 = 4.77%. The gap holds, so this is a genuine version effect, not a seasonal artifact.

**Verdict:** The latest release *regressed* signup conversion — down ~1.3 points (6.11% → 4.77%, roughly a 22% relative drop). Worth flagging to the mobile team for a rollback or investigation.
